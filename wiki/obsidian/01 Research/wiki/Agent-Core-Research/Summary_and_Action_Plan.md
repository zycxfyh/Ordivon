# Quant-Agent 改造落地蓝图：从 claw-code 精华到实战应用

> 综合所有 11 篇分析，提炼出可落地的行动优先级。

---

## 一、知识全景图

```
claw-code 精华提炼
├── 安全防护层
│   ├── [01] 权限分级 → 交易确认前强制 Prompt 模式
│   └── [02] 沙箱隔离 → 未来自定义脚本的执行防护
├── 数据操作层
│   └── [03] 文件防护 → WikiService 路径穿透防御 + 大文件截断
├── 任务管理层
│   ├── [04] 任务注册表 → 长时回测后台化 + 进度追踪
│   └── [07] 故障自愈 → API 熔断三级降级策略
├── 智能对话层
│   ├── [06] 会话压缩 → 长对话 Token 成本自动管控
│   ├── [08] Prompt 缓存 → K 线上下文哈希复用
│   ├── [09] 对话引擎 → 流式事件 + Token 预算告警
│   ├── [10] 路由引擎 → 意图预分发 + Slash 命令快速通道
│   └── [12] 上下文策略 → 引入 Rewind 回溯理念，防止上下文腐烂
├── 扩展架构层
│   └── [05] MCP/LSP → 未来外部数据源接入的协议范本
└── 工作哲学层
    └── [11] 自主开发哲学 → 人机职责分层 + 事件总线解耦
```

## 二、优先级行动计划

### 🔴 P0：立即可做（不改架构）
| 行动 | 来源 | 预估工时 |
|------|------|---------|
| WikiService 增加路径穿透防御（canonicalize + startswith 校验） | 模块 01+03 | 1h |
| api_server 增加 Token 消耗日志（每次对话记录 input/output tokens） | 模块 09 | 0.5h |
| 聊天面板增加流式状态展示（"正在获取行情..." 等中间态） | 模块 09 | 2h |

### 🟡 P1：本周可做（小规模重构）
| 行动 | 来源 | 预估工时 |
|------|------|---------|
| API 三级熔断（重试 → 降级模型 → 通知用户） | 模块 07 | 3h |
| K 线上下文哈希缓存（相同市场状态复用 context，不重新发 API） | 模块 08 | 4h |
| 消息历史滑动窗口（超过 20 轮自动压缩，保持近 5 轮原文） | 模块 06 | 4h |

### 🟢 P2：下一迭代（需规划设计）
| 行动 | 来源 | 预估工时 |
|------|------|---------|
| 回测任务注册表（后台长时任务 + 进度 WebSocket 推送） | 模块 04 | 1-2天 |
| 事件总线解耦（Agent 发事件，总线负责 Dashboard/Wiki/告警） | 模块 11 | 2-3天 |
| Slash 命令快速通道（/position /close /btc 绕过 AI 路由） | 模块 10 | 1天 |
